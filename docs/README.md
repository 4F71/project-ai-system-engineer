# Docs

This directory contains the **written system** behind the repository: decisions, learning notes, weekly logs, and operational documentation.

## What belongs here?
- **Architecture**: design records and operational runbooks
- **Concepts**: structured learning notes (math/ML/system topics)
- **Journal**: weekly progress logs (deliverables + evidence)
- **Retrospectives**: periodic reflection and process improvement
- **English**: English development artifacts for interviews/writing
- **Templates**: minimal structures referenced by READMEs (public)

## How to use
- Each subfolder has its own `README.md` describing:
  - when to write
  - how to write
  - which template to use
- Templates live in `docs/templates/` and are referenced (not copied) by READMEs.

## Related outputs
Written docs should point to concrete work in:
- `projects/`
- `proofs/`
- `roadmap/`
- `scripts/`
