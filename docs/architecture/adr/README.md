# ADR (Architecture Decision Records)

Architecture Decision Records capture **context + decision + consequences**.

## When to write
- A decision changes architecture, tooling, evaluation, deployment, or data flow
- You need a permanent “why” record for future readers

## Format
Create a new ADR file named:

`YYYY-MM-DD-<short-kebab-title>.md`

Minimum sections:
- Context
- Decision
- Alternatives considered
- Consequences (trade-offs)
- Links (issues, PRs, proofs, projects)

## Related outputs
- Link to affected `projects/` components
- Link to relevant `proofs/` validating the decision
