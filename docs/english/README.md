# English Development

Artifacts to improve English for:
- async interviews
- technical writing
- short, clear communication

## What goes here
- practice logs
- rewritten answers (EN + TR if needed)
- interview Q&A sets tied to real projects/proofs

## When to write
- After an async interview practice
- After rewriting an explanation to be clearer
- When preparing for a real application/interview

## How to write
Keep outputs short and reusable:
- 1 topic = 1 file
- link to the project/proof it references

## Related outputs
- `projects/` (source material)
- `proofs/` (evidence)
