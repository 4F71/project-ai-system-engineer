# Retrospectives

Periodic review of the learning system:
- what worked
- what failed
- what to change next cycle

## When to write
- Monthly or per phase milestone
- After completing a major roadmap phase

## How to write
Use the template:
- `docs/templates/retrospective-template.md`

File naming:
- `YYYY-MM-retro.md` (e.g., `2026-02-retro.md`)

## Related outputs
- `docs/journal/` (input)
- `roadmap/` (plan updates)
- `projects/` / `proofs/` (evidence)
