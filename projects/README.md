# Projects

This directory contains **end-to-end projects** that apply concepts, decisions, and proofs into concrete systems.

Projects are not experiments or notes.  
They are **integrated systems** that demonstrate how learning turns into implementation.

## What belongs here?
- Capstone-level systems
- Multi-step pipelines
- Evaluated or reviewable implementations
- Projects that combine ML / LLM / systems thinking

## What does NOT belong here?
- Isolated experiments
- Single-concept demos
- Draft notes (those belong in `docs/` or `proofs/`)

## How projects are structured
Each project folder must include:
- a `README.md` explaining the system
- links to related proofs and docs
- clear scope and boundaries

## Related outputs
- `docs/` (design, decisions, learning context)
- `proofs/` (evidence of competence)
- `scripts/` (supporting tooling)
